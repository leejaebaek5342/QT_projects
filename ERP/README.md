# Qt로 재고 관리 시스템 만들기
  
로그인, 회원가입, 입고, 출고, 지점 추가, 재고 현황 조회, 로그 조회, 검색, 수정, 삭제가 가능한  
재고 관리 시스템을 만들어보았다.

이번 프로젝트는 단순히 상품 목록을 보여주는 수준이 아니라,  
사용자 계정별로 데이터를 구분하고, 입출고 이력을 기록하며,  
현재 재고 상태를 한눈에 확인할 수 있는 구조를 만드는 것을 목표로 진행하였다.

## 이 프로그램은 다음과 같은 흐름으로 동작한다

- 로그인 또는 회원가입
- 메인 대시보드 화면 이동
- 입고 / 출고 / 지점 추가
- 로그 테이블 기록
- 재고 현황 테이블 계산 및 출력
- 날짜, 품목번호, 품목명 기준 검색
- 검색 결과 수정 / 삭제
- CSV 내보내기
- 로그아웃 또는 프로그램 종료

## 프로그램 구성

프로그램은 크게 `로그인/회원가입 화면`과 `메인 대시보드 화면`으로 구성하였다.

로그인 화면에서는 사용자 계정을 확인하고,  
회원가입 화면에서는 이름, 아이디, 비밀번호를 입력해 계정을 생성할 수 있도록 하였다.

메인 화면에서는 입고, 출고, 지점 추가 기능을 실행할 수 있고,  
재고 현황 테이블과 로그 테이블을 함께 확인할 수 있도록 구성하였다.  
또한 검색, 수정, 삭제, CSV 내보내기 기능을 통해  
재고 데이터를 보다 편리하게 관리할 수 있도록 했다.

## 구현 방식

회원 정보는 처음에는 txt 파일로 관리했지만,  
이후에는 확장성을 고려해 `members.json` 방식으로 변경하였다.

재고 데이터는 CSV 파일을 기반으로 관리하였다.  
입고, 출고, 지점 추가가 발생하면 날짜, 작업 구분, 품목코드, 품목명, 지점명, 수량을  
한 줄의 로그 데이터로 저장하도록 구현하였다.

저장된 CSV 데이터는 두 가지 방식으로 활용된다.

- 로그 테이블: 작업 내역 그대로 출력
- 재고 현황 테이블: 입고 / 출고 데이터를 계산해 현재 재고 상태 출력

검색 기능은 날짜, 품목번호, 품목명 기준으로 필요한 기록만 확인할 수 있도록 구현하였다.  
검색 결과는 별도의 테이블에 출력되도록 구성하여  
전체 로그와 검색 결과를 구분해서 확인할 수 있도록 하였다.

수정 기능은 검색 결과 테이블에서 선택한 셀의 값을 변경하는 방식으로 구현하였다.  
다만 날짜와 작업 구분처럼 로그의 기준이 되는 값까지 수정 가능하게 두면  
데이터의 신뢰성이 떨어질 수 있다고 판단해,  
해당 열은 수정할 수 없도록 제한하였다.

삭제 기능은 선택한 로그 데이터를 제거할 수 있도록 구현하였다.  
잘못 입력된 데이터나 더 이상 필요하지 않은 기록을 정리할 수 있도록 하였고,  
삭제 이후에는 테이블을 다시 불러와 변경된 내용이 바로 반영되도록 구성하였다.

추가로 대시보드에서는  
전체 재고 합계, 일일 입고량, 일일 출고량도 함께 표시하여  
현재 재고 상태와 하루 동안의 흐름을 빠르게 확인할 수 있도록 했다.

## 문제 해결

프로젝트를 진행하면서 먼저 부딪힌 문제는  
입고, 출고, 지점 추가 과정에서 중복되거나 충돌하는 데이터가 그대로 저장될 수 있다는 점이었다.

예를 들어 같은 품목번호에 다른 품목명이 입력되거나,  
같은 품목명에 다른 품목번호가 입력되는 경우는 반려되어야 했다.  
또 이미 존재하는 지점명이 다시 추가되는 경우에도  
중복 지점이 생성될 수 있는 문제가 있었다.

이 문제를 해결하기 위해  
기존 CSV 데이터를 먼저 확인한 뒤,  
품목번호, 품목명, 지점명이 충돌하거나 중복되는 경우  
입력을 저장하지 않고 경고 메시지를 출력하도록 수정하였다.

또 다른 문제는 로그 테이블 정렬 방식이었다.  
처음에는 오래된 로그가 위쪽에 표시되었지만,  
사용자는 보통 최근 작업 내역을 먼저 확인하기 때문에  
이 구조는 사용성이 떨어진다고 판단하였다.

그래서 CSV에서 읽어온 로그 데이터를 화면에 출력하기 전에 순서를 뒤집어  
최신 데이터가 항상 위에 오도록 수정하였다.  
이후에는 최근 입고, 출고, 지점 추가 기록을 더 빠르게 확인할 수 있게 되었다.

마지막으로 지점 삭제와 관련된 문제도 있었다.  
지점 추가 로그를 삭제해도,  
해당 지점에서 발생한 입고 / 출고 로그가 남아 있으면  
재고 현황 테이블에서 그 지점 컬럼이 계속 유지되는 문제가 발생하였다.

원인을 확인해보니  
재고 현황 테이블의 지점 목록이 지점 추가 로그만이 아니라  
전체 로그 데이터를 기준으로 계산되고 있었기 때문이었다.

이를 해결하기 위해  
삭제 대상이 지점 추가 로그인 경우에는  
해당 지점명과 관련된 입고 / 출고 로그도 함께 삭제하도록 수정하였다.  
이후에는 지점을 삭제했을 때 재고 현황에서도 정상적으로 반영되도록 정리할 수 있었다.

## 리팩토링

프로젝트 후반부에는  
MainWindow 안에 여러 로직이 몰려 있던 구조를 정리하기 위해 리팩토링을 진행하였다.

- `DataManager` : CSV 입출력 및 재고 계산
- `SessionManager` : 사용자 세션 및 파일 경로 관리
- `TableManager` : QTableWidget 설정 및 UI 코드 정리

이렇게 역할을 분리하면서 코드 가독성과 유지보수성을 높이고자 했다.

## 느낀점

이번 프로젝트를 하면서  
기능 하나를 만드는 것보다도,  
이미 만든 기능들이 서로 충돌하지 않게 유지하는 일이 더 어렵다는 걸 느꼈다.
 
또 처음에는 기능이 돌아가는 것 자체에 집중했는데,  
점점 기능이 늘어나면서  
코드를 나누지 않으면 나중에 스스로 다시 읽기도 어려워진다는 점도 많이 느꼈다.  
그래서 이번 프로젝트는 단순히 결과물을 만드는 것뿐 아니라,  
기능을 추가할수록 구조 정리가 왜 중요한지 직접 체감한 경험이기도 했다.
  
직접 화면을 구성하고 데이터를 관리하고 예외 상황을 처리해보면서  
Qt 프로그램을 조금 더 실제 프로그램답게 만드는 연습을 할 수 있었던 점에서 의미가 있었다.
