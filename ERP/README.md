# Qt 기반 재고 관리 시스템

## Overview
Qt Widgets와 C++을 사용해 구현한 재고 관리 시스템입니다.  
사용자 계정별 데이터 관리, 입출고 로그 기록, 현재 재고 계산, 검색/수정/삭제 기능을 포함합니다.

## 기술 스택
- Language: C++
- Framework: Qt Widgets
- Data: JSON, CSV

## 이 프로그램은 다음과 같은 흐름으로 동작한다

- 로그인 / 회원가입을 통해 사용자 인증
- 메인 대시보드 진입 후 기능 선택
- 입고 / 출고 / 지점 추가 수행
- 모든 작업은 CSV 로그 데이터로 기록
- 로그 데이터를 기반으로 재고 현황을 계산하여 테이블에 출력
- 날짜, 품목번호, 품목명 기준으로 로그 검색
- 검색 결과에 대해 수정 및 삭제 수행
- 데이터 CSV 파일로 내보내기
- 로그아웃 또는 프로그램 종료

## 프로그램 구성

프로그램은 `로그인/회원가입 영역`과 `메인 대시보드/데이터관리 영역`으로 구분하여 설계하였다.

- 로그인/회원가입 영역  
  사용자 인증 및 계정 생성을 담당하며, 사용자별 데이터 분리를 위한 기반을 구성한다.

- 메인 대시보드 영역  
  입고, 출고, 지점 추가와 같은 재고 관리 기능을 수행하며,  
  로그 테이블과 재고 현황 테이블을 통해 데이터의 입력과 결과를 동시에 확인할 수 있도록 구성하였다.

- 데이터 관리(검색/수정/삭제) 영역
  검색, 수정, 삭제 기능을 통해 재고 데이터를 보다 편리하게 관리할 수 있도록 했다.
  
또한 전체 구조는 다음과 같이 역할을 분리하였다.

- DataManager: CSV 데이터 처리 및 재고 계산 로직 담당  
- SessionManager: 사용자 세션 및 파일 경로 관리  
- TableManager: UI 테이블 구성 및 출력 처리


## 구현 방식

### 데이터 저장

- 회원 정보는 초기에는 txt 파일로 관리했지만, 확장성을 고려해 `members.json` 방식으로 변경하였다.
- 재고 데이터는 CSV 파일을 기반으로 관리하였다.
- 입고, 출고, 지점 추가가 발생하면 날짜, 작업 구분, 품목코드, 품목명, 지점명, 수량을 한 줄의 로그 데이터로 저장하였다.

### 데이터 활용

저장된 CSV 데이터는 다음 두 가지 방식으로 활용된다.

- 로그 테이블: 작업 내역을 그대로 출력
- 재고 현황 테이블: 입고 / 출고 데이터를 계산하여 현재 재고 상태 출력

### 검색 / 삭제 / 수정

- 날짜, 품목번호, 품목명 기준으로 필요한 로그를 검색할 수 있도록 구현하였다.
- 검색 결과는 별도 테이블에 출력하여 전체 로그와 구분하였다.
- 검색 결과에서 선택한 데이터를 수정하거나 삭제할 수 있도록 구현하였다.
- 날짜와 작업 구분은 로그의 기준값이므로 수정할 수 없도록 제한하였다.

### 대시보드 요약 정보

- 전체 재고 합계
- 일일 입고량
- 일일 출고량

위 정보를 대시보드에 표시하여 현재 재고 상태와 하루 동안의 흐름을 빠르게 확인할 수 있도록 하였다.

## 문제 해결

### 1. 데이터 중복 및 충돌 문제

- 문제  
  동일한 품목번호에 다른 품목명이 입력되거나,  
  동일한 품목명에 다른 품목번호가 입력되는 문제가 발생하였다.  
  또한 동일한 지점명이 중복 생성될 수 있었다.

- 해결  
  기존 CSV 데이터를 검사하여  
  품목번호, 품목명, 지점명이 충돌하거나 중복되는 경우  
  입력을 저장하지 않고 경고 메시지를 출력하도록 수정하였다.

---

### 2. 로그 정렬 방식 문제

- 문제  
  오래된 로그가 상단에 표시되어 최근 작업 내역 확인이 불편하였다.

- 해결  
  CSV 데이터를 읽은 후 출력 전에 순서를 뒤집어  
  최신 데이터가 항상 상단에 표시되도록 수정하였다.

---

### 3. 지점 삭제 시 데이터 정합성 문제

- 문제  
  지점 추가 로그를 삭제해도 해당 지점의 입고/출고 로그가 남아  
  재고 현황 테이블에 지점 컬럼이 계속 유지되는 문제가 발생하였다.

- 해결  
  지점 추가 로그 삭제 시 해당 지점과 관련된  
  입고/출고 로그도 함께 삭제하도록 수정하였다.

## 리팩토링

프로젝트 후반부에는  
MainWindow 안에 여러 로직이 몰려 있던 구조를 정리하기 위해 리팩토링을 진행하였다.

- `DataManager` : CSV 입출력 및 재고 계산
- `SessionManager` : 사용자 세션 및 파일 경로 관리
- `TableManager` : QTableWidget 설정 및 UI 코드 정리

이렇게 역할을 분리하면서 코드 가독성과 유지보수성을 높이고자 했다.

## 느낀점

이번 프로젝트를 하면서  
기능 하나를 만드는 것보다도,  
이미 만든 기능들이 서로 충돌하지 않게 유지하는 일이 더 어렵다는 걸 느꼈다.
 
또 처음에는 기능이 돌아가는 것 자체에 집중했는데,  
점점 기능이 늘어나면서  
코드를 나누지 않으면 나중에 스스로 다시 읽기도 어려워진다는 점도 많이 느꼈다.  
그래서 이번 프로젝트는 단순히 결과물을 만드는 것뿐 아니라,  
기능을 추가할수록 구조 정리가 왜 중요한지 직접 체감한 경험이기도 했다.
  
직접 화면을 구성하고 데이터를 관리하고 예외 상황을 처리해보면서  
Qt 프로그램을 조금 더 실제 프로그램답게 만드는 연습을 할 수 있었던 점에서 의미가 있었다.
